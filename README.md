# FocusSpark Chrome Extension

FocusSpark is an AI-powered study and productivity product. This folder contains the Chrome extension half of the frontend.

The extension is focused on the in-browser study experience: opening FocusSpark from Chrome, working inside a focused study workspace, using AI chat, generating flashcards and quizzes, tracking attention, and enforcing strict mode while the user studies.

The other half of the frontend will live on the website. Website pages such as profile, settings, account management, about pages, and larger summary/analyze dashboards should be handled there instead of being treated as the main responsibility of this extension folder.

## Scope

This `FocusSpark-Extension` folder includes:

- Chrome Extension Manifest V3 setup
- Extension launcher and background service worker
- React app rendered inside the extension page
- Sign in, sign up, and onboarding flow
- Student dashboard and study workspace
- AI chatbot workspace
- Chat history view
- Flashcard deck screen
- Quiz flow
- Focus detection UI and webcam/focus testing screens
- Pomodoro and focus mode context
- Strict mode tab control through the Chrome extension background script
- Backend integration for chat, flashcards, quizzes, analyze, and WebSocket focus updates

The separate website frontend should own:

- Full user profile pages
- Full settings portal
- Account and preference management
- About us, marketing, and public website pages
- Large summary/analyze dashboards
- Broader website navigation and non-extension pages

Ignore the temporary `chrome-frontend` folder when working on this part of the product. The source of truth for the extension frontend is this `FocusSpark-Extension` folder.

## Project Structure

```text
FocusSpark-Extension/
  public/
    manifest.json       # Chrome extension manifest
    background.js       # MV3 service worker for launch + strict mode tab control
    icon*.png           # Extension icons
  src/
    App.tsx             # Main React page routing/state
    main.tsx            # React entry point
    config/backend.ts   # Backend base URL and route constants
    context/            # Focus and Pomodoro providers
    utils/              # Backend/auth helpers
    components/         # Extension screens and reusable UI
  build/                # Production extension output generated by npm run build
```

## Requirements

- Node.js
- npm
- Google Chrome or a Chromium-based browser
- FocusSpark backend running locally at `http://127.0.0.1:8000`

The extension currently expects backend HTTP and WebSocket access at:

- `http://127.0.0.1:8000/*`
- `ws://127.0.0.1:8000/*`

These hosts are already listed in `public/manifest.json`.

## Local Development

Install dependencies:

```bash
npm install
```

Run the Vite development server:

```bash
npm run dev
```

The dev server is configured for port `3000`.

## Build

Create the production extension bundle:

```bash
npm run build
```

The compiled extension is written to:

```text
build/
```

## Load in Chrome

1. Run `npm run build`.
2. Open `chrome://extensions`.
3. Turn on Developer mode.
4. Click Load unpacked.
5. Select the `FocusSpark-Extension/build` folder.

After changing `public/manifest.json` or `public/background.js`, rebuild the project and reload the extension from `chrome://extensions`.

## Extension Behavior

Clicking the FocusSpark extension icon opens the extension app in a new Chrome tab.

Strict mode is managed by `public/background.js`. When enabled, the background service worker tracks the focus tab, detects tab switching or new distraction tabs, sends runtime events back to the app, and can close newly opened distraction tabs during protected focus sessions.

Auth tokens are stored in Chrome extension storage under:

```text
fs_access_token
```
